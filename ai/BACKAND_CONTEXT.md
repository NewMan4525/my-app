# Архитектура проекта и Спецификация Хелперов (EVE Online Market App)

Этот документ предназначен для LLM/AI-ассистентов с целью мгновенного понимания контекста, оптимизаций производительности и доступного инструментария проекта.

## 🛠 Общий технологический стек и ограничения производительности

- **Next.js 16+ (App Router)**, **React 19**, **TypeScript 5+**.
- **База данных**: `better-sqlite3`. Работает в синхронном режиме. Использует **одно постоянное соединение** на протяжении жизни сервера. SQL-запросы (Prepared Statements) компилируются **строго один раз** при старте модуля базы данных (`stmtGet`, `stmtSet` и т.д.) для предотвращения деоптимизации SQLite.
- **Сетевой слой**: ESI API (Eve Tech). Запросы ограничены через `p-limit(50)` во избежание блокировки сокетов. Реализован `EsiShield` для отслеживания лимита ошибок (`x-esi-error-limit-remain`) и автоматического ухода в коулдаун на 60 секунд при ошибке 420.
- **Оптимизация V8**: Проект оптимизирован под пиковую производительность движка V8. Запрещено использовать цепочки медленных методов массивов (`.map().filter().flat()`) на больших объемах данных. Память под массивы предвыделяется заранее через `new Array(len)`. Применяются только классические итераторы `for`. Все приведения типов (`String(id)`) вынесены за пределы вложенных (горячих) циклов.

## 📂 Глобальные хелперы и утилиты (`./src/lib/helpers.ts`)

Каждая функция спроектирована как чистая, изолированная микрофункция. V8 эффективно инлайнит эти методы. При работе с кодом вызывайте их вместо написания кастомных циклов.

### 📊 Обработка массивов и данных

1. **`sortItems<T>(items: T[], key: keyof T, desc?: boolean): T[]`**
    - _Что делает_: Безопасно сортирует массив объектов по ключу, возвращая новый массив (без мутации исходного).
2. **`safeArrayMap<T, R>(items: T[], callback: (item: T, index: number) => R, context: string): R[]`**
    - _Что делает_: Оптимизированный маппинг массивов с предвыделением памяти точной длины и встроенным блоком `try-catch`.
3. **`flattenValidPages<T>(pages: (T[] | null)[]): T[]`**
    - _Что делает_: Высокопроизводительное плоское слияние страниц за один проход, полностью игнорируя `null`-ответы сети. Заменяет медленную связку `.filter().flat()`.

### 💰 Экономика и расчеты EVE Online

4. **`calculateMinMaxPrices(orders: { is_buy_order: boolean, price: number }[]): { maxBuy: number, minSell: number }`**
    - _Что делает_: За один проход по массиву ордеров вычисляет лучшую цену покупки (`maxBuy`) и лучшую цену продажи (`minSell`).
5. **`roundIsk(value: number): number`**
    - _Что делает_: Быстрое округление любой денежной суммы игровых ISK строго до двух знаков после запятой.
6. **`calculateFeeWithLimit(price: number, percent: number, minLimit: number): number`**
    - _Что делает_: Рассчитывает процентный налог брокера/SCC с учетом минимального порога (минимальный лимит 100 ISK для брокера и 25 ISK для SCC).

### 🔍 Фильтрация, парсинг и валидация

7. **`isInRange(value: number, min: number, max?: number | null): boolean`**
    - _Что делает_: Быстрая проверка вхождения числа в диапазон `[min, max]`. Если `max` не задан, проверяет только нижнюю границу. Used в атомарных фильтрах.
8. **`isValidData<T>(value: T | null | undefined): value is T`**
    - _Что делает_: Защитный Type Guard для отсечения `null` и `undefined`.
9. **`fastParseYmd(dateStr: string): number`**
    - _Что делает_: Парсит дату формата "YYYY-MM-DD" в UNIX-timestamp с помощью арифметики строк. Работает в ~10 раз быстрее встроенного `Date.parse()`.
10. **`splitCsvLine(line: string): string[]`**
    - _Что делает_: Парсер строки CSV. Не создает промежуточных микрострок на каждой итерации, сохраняя память и не нагружая Garbage Collector. Корректно обрабатывает кавычки в именах станций.
11. **`parsePairKey(pairKey: string): { typeID: number, orderType: 'buy' | 'sell' }`**
    - _Что делает_: Разбивает составной строковый ключ вида `12345-buy` на составные части.

### 🌐 Сетевой слой и инфраструктура

12. **`parseExpiresHeader(headers: Headers): number`**
    - _Что делает_: Безопасно вытягивает UNIX-timestamp времени окончания кэша из HTTP-заголовка `expires`.
13. **`getOrParseCachedData<T>(cached: ICacheData): T`**
    - _Что делает_: Реализует паттерн ленивой десериализации JSON из базы данных. Если данные уже парсились в рамках текущей сессии, отдает готовый объект из свойства `parsedData` без вызова `JSON.parse()`.
14. **`buildUrlsFromItems(baseUrl: string, items: { type_id: number }[]): string[]`**
    - _Что делает_: Высокопроизводительный пакетный конструктор URL. Конкатенирует базовый URL с массивом `type_id` предметов.
15. **`safeUnlinkSync(filePath: string): void`**
    - _Что делает_: Безопасное удаление технических файлов базы данных SQLite (включая `-wal` и `-shm`) с диска.

## 📐 Архитектура Слой-за-Слоем

1. **`dbHandlers.ts` / `db.ts`**: Слой хранения. Кэширует сырые ответы ESI API. Прагмы WAL-режима выкручены на максимум (`synchronous = NORMAL`).
2. **`querysHandler.ts`**: Слой получения данных. Оркестрирует запросы. Сначала проверяет валидность локального кэша, при просрочке шлет запрос с заголовком `If-None-Match`. Если получает `304 Not Modified`, просто продлевает жизнь записи, экономя трафик.
3. **`dataHandlers.ts`**: Первичная агрегация данных. Группирует ордера, считает объемы истории.
4. **`filters.ts`**: Слой поэтапного отсечения. Фильтры (цена, маржа, прыжки) намеренно разделены и выполняются **между** сетевыми запросами. Это ключевая фича: отсечь ненужные `type_id` как можно раньше, чтобы сервер не делал тысячи тяжелых HTTP-запросов к истории и именам предметов.
5. **`profitCalculations.ts`**: Бухгалтерский слой. Рассчитывает чистую прибыль, налоги (NPC Брокер, Цитадели, Accounting, Standings). Отсекает убыточные позиции на лету через `continue`.
6. **`execute.ts`**: Главный конвейер (Pipeline) бэкенда для общего анализа рынка.
7. **`warExecuter.ts` / `warUtils.ts`**: Конвейер анализа конкуренции на основе загруженных пользователем файлов `marketlogs.csv`.
