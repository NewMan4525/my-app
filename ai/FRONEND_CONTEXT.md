# Спецификация и Архитектура Фронтенда (ISK MASTER)

Этот документ предназначен для LLM/AI-ассистентов с целью мгновенного понимания реактивной архитектуры, структуры хуков, пресетов и правил оптимизации клиентской части Next.js.

## 🏛️ Ключевые принципы и ограничения фронтенда

- **Next.js 16 (App Router)**, **React 19**, **Tailwind CSS v4** [1]. Strict TypeScript (без использования `any`).
- **MPA (Multi-Page Application) Навигация**: Переходы между страницами осуществляются строго через нативные теги `<a>` (компонент `@/src/components-generic/link`). Тяжёлые глобальные реактивные стейты запрещены. Переход полностью сбрасывает состояние React-компонентов текущей страницы.
- **Блокировка каскадных ререндеров**: Запрещено вызывать `setState` внутри синхронного потока выполнения `useEffect` (защита от Hydration Mismatch и паразитных циклов перерисовки). Чтение `localStorage`/`sessionStorage` при инициализации стейта изолируется с помощью динамического импорта компонентов `{ ssr: false }`. Восстановление данных в `useEffect` оборачивается в `setTimeout(..., 0)`.
- **0-ms Direct DOM Mutations (Правило WAR PAGE)**: Изменение визуального состояния строк и ячеек таблиц при быстрых действиях (копирование, чекбокс «выполнено», переключение статусов OUTBID/IGNORED) выполняется **напрямую через DOM** (`e.currentTarget.textContent`, `row.style.opacity = '0.35'`) в обход стейта React. Это гарантирует нулевую задержку интерфейса и защищает таблицы от полной перерисовки.

## 📂 Структура папок и Модули

### 📊 1. Глобальные Фронтенд-Хелперы (`@/src/utils/helpers.ts`)

Чистые изолированные утилиты, оптимизированные для инлайнинга движком V8.

- **`calculateModifiedIsk(price, direction)`**: Математический расчёт модификации цены по правилу 4 значащих цифр EVE Online с шагом `±1` в наименьшем значащем разряде.
- **`copyToClipboardWithFeedback(button, text, className)`**: Высокопроизводительный UI-хелпер. Копирует текст в буфер обмена и мгновенно мутирует текст кнопки на `[Copied]` с защитой от спам-кликов (`pointerEvents = 'none'`) без ререндера таблицы.
- **`getFromStorage / setToStorage`**: Безопасные обертки над LocalStorage с защитой от ошибок сборки на этапе серверного рендеринга (SSR Check).
- **`getFromSession / setToSession`**: Обертки над SessionStorage. Используются для кэширования серверных ответов (`cached_market_items`) для мгновенного 0-ms восстановления данных при навигации по вкладкам.

### ⚙️ 2. Кастомные Хуки-Контроллеры (`@/src/utils/hooks/`)

Вся асинхронная и фоновая логика страниц изолирована здесь.

- **`useDomRadar.ts`**: Автономный сканер DOM-дерева. Проверяет наличие строк со статусом `OUTBID` через чистый цикл `for`. Управляет интервалом мигания вкладки браузера `document.title` (`🔴 0.01 ISK WAR`) и триггерит нативные системные пуш-уведомления Windows через `Notification API` строго один раз за тик данных (используя реф-предохранитель `hasAlertedThisTickRef`).
- **`useChromiumWorker.ts`**: Изолирует жизненный цикл фонового `Worker`, создаваемого динамически через `Blob URL`. Обходит жесткий троттлинг Chromium для фоновых вкладок. Стабильно тикает каждые `5m 05s` (окно сброса кэша CCP Games) и автоматически пушит сохраненный текст лога на `/api/war` через `fetch`, обновляя таблицы.

### 📝 3. Статические Конфигурационные Пресеты (`@/src/utils/presets/`)

Вынесение громоздких конфигураций инпутов из тела компонентов для разгрузки стека V8 и читаемости кода.

- **`marketOptions.ts`**: Генерирует массив экземпляров `InputsBlockOptionCreator` для числовых лимитов рынка и радио-групп торговых хабов Житы/Цитаделей.
- **`userOptions.ts`**: Формирует структуру полей для стендингов фракций/корпораций и уровней социальных навыков (1–5) персонажа.

### 🏭 4. Оптимизированная Фабрика Форм (`@/src/components-feature/`)

- **`inputsBlock.tsx`**: Контейнер-распределитель блоков полей ввода.
- **`formInputItem.tsx`**: Изолированная фабрика инпутов (`switch-case`). Обернута в умный `React.memo` с кастомной функцией сравнения пропсов. **Критично**: блокирует перерисовку всей формы; при вводе символов в инпут цены, перерисовывается строго этот инпут, а соседние радиокнопки и элементы формы React полностью игнорирует.

### 🗂️ 5. Универсальный Табличный Компонент (`@/src/components-generic/table.tsx`)

Высокопроизводительный generic-компонент `<Table<T>>`, защищенный от нагрузок.

- **Windows-style Multi-select**: Поддерживает выделение строк через `Ctrl` (точечно) и `Shift` (сплошной диапазон с очисткой старого выделения за его пределами).
- **Anti-Blue Highlight Fix**: При зажатом `Shift` вешает обработчик на уровень `window` и блокирует нативное выделение текста синим цветом через `e.preventDefault()` и `document.body.style.userSelect = 'none'`.
- **CPU Optimisation**: Тяжёлый строковый метод `localeCompare` внутри `useMemo` сортировки полностью удалён и заменён на быстрые логические операторы сравнения примитивов `>` и `<`.

## 🚨 Инструкция для ИИ по генерации нового UI-кода

1. **Импорты**: Все пути к утилитам должны использовать алиас `@/src/utils/helpers`. Запрещено импортировать старые удалённые файлы `storage` или `clipboardModify`.
2. **Контролируемые формы**: Все радиокнопки и числовые поля должны управляться через связку `value/checked` + колбэки `onChange` из верхнего стейта страницы. Никакого неуправляемого дребезга.
3. **Запрет на `any`**: Любые новые массивы данных для таблиц или пропсы должны иметь жёсткие типы или расширять базовые интерфейсы `IMarketItem`, `IWarItem`, `IInputProps`.
4. **DOM над стейтом**: Если добавляется визуальный элемент индикации (мигание, скрытие полей), приоритет отдаётся CSS псевдоклассам (например, `:empty` для скрытия пустых кнопок статуса) или прямым DOM-мутациям элементов через события `e.currentTarget`.
