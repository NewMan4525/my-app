# === КОНТЕКСТ ПРОЕКТА: ISK MASTER (NEXT.JS APP ROUTER + TYPESCRIPT) ===

## 1. СУТЬ И АРХИТЕКТУРА ПРОЕКТА

- Проект является фронтендом для торгового симулятора / калькулятора вселенной EVE Online.
- Архитектура построена по строгому принципу Multi-Page Application (MPA).
- Логика работы: Пользователь настраивает параметры в формах, нажимает кнопку "OK" (сохранение в LocalStorage). По кнопке "Get Data" клиент собирает все три объекта настроек (trade_settings, user_stats, user_skills) и отправляет их на сервер POST-запросом в формате JSON (/api/buy).
- Никакой динамики или реактивности "на лету" после получения данных от сервера на клиенте нет (кроме смены темы, сортировки и выделения строк таблицы). Сервер возвращает полностью готовый массив просчитанных данных для таблицы ордеров (тип IMarketItem).
- Проект имеет жесткие правила линтинга (strict: true): ПОЛНОСТЬЮ ЗАПРЕЩЕН тип "any" (везде используется строгая типизация интерфейсов) и ЗАПРЕЩЕН синхронный "setState" внутри "useEffect" во избежание каскадных рендеров и ошибок гидратации.

## 3. ТЕКУЩИЙ СТАТУС И РЕАЛИЗОВАННЫЙ ФУНКЦИОНАЛ

- **Инициализация стейта**: Локальные стейты форм имеют плоскую структуру Record<string, number | string>. Они лениво инициализируются из localStorage, а если его нет — берут дефолты бэкенда.
- **Делегирование событий на странице Buy**: На теге <section> страницы buy/page.tsx установлен единый менеджер кликов handlePageClick. Он перехватывает клики по кнопкам по атрибутам data-action.
- **Паттерн Оверлея (Overview + OptionBar)**: Компонент Overview реализован как переиспользуемый контейнер со строгой типизацией props.children. При открытии настроек (isOptionsVisible === true) он рендерит внутри себя форму OptionBar и инфо-панель InfoPanelNoSSR.
- **Логика выключения InfoPanel**: Когда оверлей открыт, в основном потоке страницы под Header инфо-панель возвращает null, освобождая место, и таблица ордеров поднимается на самый верх к шапке.
- **Потокобезопасная изоляция бэкенда**: Реализован Вариант А. Функции фильтрации и расчета прибыли полностью отвязаны от глобальных изменяемых констант. Все входящие параметры (activeSettings, activeStats, activeSkills) прокидываются изолированно через аргументы функций в рамках текущего POST-запроса.
- **Оптимизация кэша SQLite**: Модуль dbHandlers.ts удерживает одно открытое соединение на протяжении жизненного цикла процесса Node.js. Включен высокопроизводительный режим WAL (Write-Ahead Logging). Автоматическое создание папки /src/db/ и таблиц происходит при первом обращении к модулю.
- **Защита UI от дублирующих запросов**: При нажатии на кнопку "Get Data" в InfoPanel, задействуется хук useTransition. Кнопка получает атрибут disabled и меняет текст на "...Loading", предотвращая повторную отправку тяжелых запросов.
- **Универсальный модуль Table**: Реализована дженерик-таблица `Table<T>` с динамической конфигурацией колонок через пропсы, встроенной клиентской сортировкой (с яркой неоновой подсветкой активной колонки `▲`/`▼`) и сверхкомпактной версткой Excel-style (`white-space: nowrap`, `tabular-nums`).
- **Продвинутое выделение строк**: Внутри модуля таблицы реализована Windows-механика множественного выделения строк. Обычный клик сбрасывает стейт, **Ctrl + Клик** точечно добавляет/удаляет строку, а **Shift + Клик** выделяет диапазон строк от последней выбранной до текущей.
- **Умное копирование (EVE Market Rules)**: Кнопки `[Copy]` встроены прямо внутрь ячеек `buy` и `Name`. Клик по ним изолирован через `e.stopPropagation()` и не сбивает выделение строк. Для цен задействован алгоритм `clipboardModify.ts`, который высчитывает порядок числа и изменяет строго 4-ю значащую цифру ордера для мгновенного перебития конкурентов в игре, зануляя лишние хвосты округления JS. Реализован мгновенный текстовый фидбек `[Copied]` через нативный DOM без каскадных ререндеров страницы.
